# a11y-delta v0.2.0 — Multi-Page Coverage Design

**Date:** 2026-06-12
**Status:** Approved
**Milestone:** v0.2.0

---

## Problem

v0.1.0 diffs a single URL pair. Real sites have dozens of pages; a PR might break `/checkout/` while leaving `/` clean. Without multi-page coverage the tool misses cross-page regressions and teams must run separate commands per page.

---

## Goals

- Audit N pages in one command, report only new violations per page
- Backward-compatible: `a11y-delta --baseline url --candidate url` unchanged
- All 4 URL sources: config file, URL file/inline list, sitemap, single-pair fallback
- Configurable concurrency (default 3 parallel browsers)
- Both per-page and combined snapshot storage
- Terminal output: Warm Studio colours, per-page tree sections
- GitHub comment: summary table + collapsible `<details>` per failing page

---

## What Does Not Change

- `src/diff.js` — `expand`, `violationKey`, `diff` unchanged
- `src/audit.js` — `auditUrl`, `buildAuditConfig` unchanged
- `src/snapshot.js` — `readSnapshot`, `writeSnapshot`, `makeSnapshot`, `validateSnapshot` unchanged (single-URL paths)
- v0.1.0 single-URL CLI flags unchanged; all existing integration tests must still pass

---

## New Files

| File | Responsibility |
|---|---|
| `src/config.js` | Load + validate `.a11y-delta.yml`; merge with CLI args (CLI wins) |
| `src/pages.js` | Resolve final page list from all 4 sources in precedence order |
| `src/sitemap.js` | Fetch + parse `sitemap.xml` via `node:http`/`node:https`; return `string[]` |
| `src/pool.js` | Concurrency pool — run N async tasks, at most M concurrent; pure ESM, no dep |
| `src/multi.js` | Orchestrate: resolve pages → pool → per-page diff → MultiResult aggregate |

## Config File — `.a11y-delta.yml`

Optional. Loaded from CWD unless `--config <path>` overrides. CLI flags always win.

```yaml
# .a11y-delta.yml
base:           https://staging.example.com   # base URL for relative page paths
candidate-base: https://preview.example.com   # candidate base URL

pages:
  - /                          # relative — resolved against base
  - /products/
  - /checkout/
  - https://other.example.com  # absolute — used as-is

concurrency: 3                 # max parallel Playwright instances (default: 3)
fail-on: critical,serious      # same as --fail-on flag (default: critical,serious)

# Snapshot storage (optional)
save-dir: .a11y-snapshots/     # per-page baseline directory
```

---

## URL Source Precedence (highest → lowest)

1. `--sitemap <url>` — auto-discovers all pages from sitemap.xml
2. `--urls <file-or-list>` — explicit list; file path OR comma-separated inline values (auto-detected)
3. `.a11y-delta.yml` `pages:` list
4. `--baseline` / `--candidate` pair — v0.1.0 single-URL mode (no multi-page)

---

## Internal Shapes

### PageResult

```javascript
{
  url:          string,           // absolute candidate URL
  baselineUrl:  string,           // absolute baseline URL (may equal url)
  newViolations: Entry[],         // from diff()
  baselineCount: number,
  candidateCount: number,
  error:        string | null,    // set if audit threw; page counted as skipped
}
```

### MultiResult

```javascript
{
  pages:      PageResult[],
  totalNew:   number,
  byCritical: number,
  bySerious:  number,
  byModerate: number,
  byMinor:    number,
  cleanPages: number,
  failPages:  number,
  errorPages: number,
  exitCode:   0 | 1 | 2,
}
```

## CLI Surface — New Flags

```
Multi-page inputs (any one, in precedence order):
  --sitemap     <url>         Fetch sitemap.xml and audit all discovered pages
  --urls        <file|list>   File path (one URL per line) or comma-separated list
  --base        <url>         Base URL for resolving relative page paths
  --candidate-base <url>      Candidate base URL (required for multi-page URL mode)

Multi-page options:
  --concurrency <n>           Max parallel browsers (default: 3)
  --config      <file>        Config file path (default: .a11y-delta.yml in CWD)
  --save-dir    <dir>         Save per-page candidate snapshots to directory
  --output-style <style>      per-page (default) | failures-only
                              (per-page = show all pages; failures-only = skip clean pages)

Existing flags (unchanged):
  --baseline, --candidate, --fail-on, --format, --save, --timeout, --viewport,
  --wait-for, --header, --help
```

### Usage examples

```bash
# Config-driven (zero extra flags after .a11y-delta.yml is set up)
a11y-delta --candidate-base https://preview-123.example.com

# Inline URL list
a11y-delta \
  --base https://staging.example.com \
  --candidate-base https://preview.example.com \
  --urls "/,/products/,/checkout/"

# URL file
a11y-delta \
  --base https://staging.example.com \
  --candidate-base https://preview.example.com \
  --urls pages.txt

# Sitemap auto-discovery
a11y-delta \
  --sitemap https://staging.example.com/sitemap.xml \
  --candidate-base https://preview.example.com

# Save per-page baseline snapshots
a11y-delta \
  --candidate-base https://staging.example.com \
  --urls pages.txt \
  --save-dir .a11y-snapshots/

# Diff candidate URLs against saved per-page snapshots
a11y-delta \
  --baseline .a11y-snapshots/ \
  --candidate-base https://preview.example.com \
  --urls pages.txt

# v0.1.0 single-URL (unchanged)
a11y-delta \
  --baseline https://staging.example.com \
  --candidate https://preview.example.com
```

## Data Flow

```
CLI args + .a11y-delta.yml
        │
        ▼
config.js  →  mergedConfig
        │
        ▼
pages.js   →  resolvePages(mergedConfig)  →  { baselineUrl, candidateUrl }[]
        │
        ▼
pool.js    →  concurrency(N, tasks)
             each task:
               auditUrl(baselineUrl) OR readSnapshot(baselineFile)  →  ViolationSet
               auditUrl(candidateUrl) OR readSnapshot(candidateFile) →  ViolationSet
               diff(baseline, candidate)  →  PageResult
        │
        ▼
multi.js   →  aggregate(PageResult[])  →  MultiResult
        │
        ▼
[--save-dir]  writeSnapshotDir(candidateViolationSets, dir)
        │
        ▼
render.js  →  renderTableMulti / renderGithubCommentMulti / renderJsonMulti
        │
        ▼
stdout  +  process.exit(MultiResult.exitCode)
```

## Snapshot Storage

### Per-page directory (`--save-dir .a11y-snapshots/`)

One file per page, named by URL path:

```
.a11y-snapshots/
  index.json                   # URL: /
  products.json                # URL: /products/
  checkout.json                # URL: /checkout/
  _manifest.json               # { base, createdAt, pages: [{url, file}] }
```

Naming: `path.split('/').filter(Boolean).join('-') || 'index'` + `.json`.

### Combined file (`--save baseline.json`)

```json
{
  "base": "https://staging.example.com",
  "timestamp": "2026-06-12T10:00:00.000Z",
  "pages": {
    "https://staging.example.com/": { ...ViolationSet },
    "https://staging.example.com/products/": { ...ViolationSet }
  }
}
```

## Error Handling

- Page audit throws (Playwright error, network timeout) → `PageResult.error = err.message`, violations = [], not counted in totals; stderr warning per page
- All pages error → exit 2, stderr "All N pages failed to audit"
- Sitemap fetch fails → exit 2, stderr "Could not fetch sitemap at <url>: <reason>"
- Config file present but invalid YAML → exit 2, stderr "Invalid .a11y-delta.yml: <reason>"
- `--urls` file not found → exit 2, stderr "URLs file not found: <path>"
- `--concurrency` < 1 or non-integer → exit 2, stderr "concurrency must be a positive integer"
- No page source specified and no config file → fall through to v0.1.0 single-URL validation (existing `--baseline`/`--candidate` required check)

## Testing Approach

- `src/config.js` — unit tests: valid yaml, missing file (ok, returns {}), invalid yaml (throws), CLI override wins
- `src/pages.js` — unit tests: each source type, precedence order, relative→absolute resolution
- `src/sitemap.js` — unit tests with mock HTTP server (same pattern as prometheus.js tests)
- `src/pool.js` — unit tests: concurrency cap (max N tasks active at once), error in one task doesn't block others
- `src/multi.js` — integration tests using fixture snapshots (no real Playwright), assert MultiResult shape
- `src/render.js` — snapshot tests for `renderTableMulti`, `renderGithubCommentMulti`, `renderJsonMulti`
- `src/snapshot.js` — unit tests for `readSnapshotDir`, `writeSnapshotDir`, `readMultiSnapshot`, `writeMultiSnapshot`
- `tests/integration.test.js` — CLI multi-page tests using fixture snapshot directories
